## 모델 종류

모델은 Checkpoint, Lora 등 여러 모델이 존재. stable-diffusion-webui/models 폴더내 저장한다.

Checkpoint의 경우 stable-diffusion 폴더에 둔다. 

`Checkpoint`의 경우 크기가 `Lora`보다 매우 큰데, `Lora`는 Low-Rank Adaptation 으로 파인튜닝된 모델이기 때문입니다.

`Lora`모델을 이용할 경우 다음과 같이 프롬프트에 문구로 들어간다. `:1`의 경우 해당 모델을 있는 그대로 반영한다는 것이고 10%의 경우는 `0.1`로 적용한다.

`Embeddings`프롬프트 파일의 경우 stable-diffusion-webui/Embeddings 폴더에 저장하고 webui 에서는 text inversion탭에서 선택하여 적용합니다.

이 프롬프트 파일의 경우 프롬프트의 모음을 한 단어로 축약하여 불러옵니다.

```text
<lora:gfsowon-v1d:1>
```

### UI 설명

`프롬프트`에는 생성할 이미지를 텍스트로 쉼표로 구분해 표현한다.

`네거티브 프롬프트`는 이미지에서 제외할 요소를 텍스트로 표현한다.

`Sampling steps`는 블러를 걷어내는 횟수

`Restore faces`는 잘못된 얼굴을 고치는데 쓰지만 잘 쓰지는 않음.

`Tileg`은 한 이미지를 타일 찍듯이 빈 공간 모두 채움

`Batch count` 총 생성할 이미지 갯수

`Batch size` 카운트당 생성할 이미지

`CFG scale` 은 프롬프트를 얼마나 잘 고정하여 만들것인지 설정 옵션에 숫자를 높일 수록 프롬프트에 적합한 이미지 생성

`Seed` 는 시드로, 이미지를 생성한 후 해당 시드로 계속해서 이미지 작업을 하게 되면 비슷한 결과를 보여줌
