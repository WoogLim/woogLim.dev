## 알고리즘의 풀이 순서

---

- 문제 해결을 위해 다음과 같은 과정이 수행되어야한다.
  1. I. 문제의 이해.
  2. II. 구체적 예시를 탐색한다.
  3. III. 문제를 세분화한다.
  4. IV. 단순한 문제 풀이.
  5. V. 리펙터.

### I. 문제의 이해

---

- 문제의 이해를 위해 면접관에게 다음과 같은 질문을 한다.
  1. I. 이 문제의 핵심 키워드는 무엇인가?
  2. II. 이 문제의 입력값은 무엇인가?
  3. III. 출력값의 형태는 무엇인가?
  4. IV. 입력값이 출력값을 결정할만한 충분한 정보를 가지고 있는가? 부수적인 정보가 필요하진 않은가.
  5. V. 변수명은 어떻게 정할것인가.

#### 풀이 예시

---

- 문제 풀이 이전에 알고리즘 풀이 순서 I ~ V 에 대한 답변을 정리한다.

```TEXT
ex) 두 수를 받아 더하는 함수 구하기

1. 이 문제의 핵심 키워드는 무엇인가? (I. 문제의 핵심 키워드를 파악한다.)
✅ 합을 구하는 함수

2. 이 문제의 입력값은 무엇인가?
✅ 두개의 입력값을 가진다. 단, 정수인지 소수인지 범위는 어디까지인가 int?, float?, large number?
(Javascript 의 경우 더할 값이 크다면 어느 시점에서 계산을 하지 않고 Infinity를 출력한다.)

3. 출력값의 형태는 무엇인가?
✅ 정수 혹은 부동소수점으로 출력하는가 int?, float?, string?

4. 입력값이 출력값을 결정할만한 충분한 정보를 가지고 있는가? 부수적인 정보가 필요하진 않은가.
✅ 두 입력이 아닌 하나의 입력인 경우 무엇을 더해 출력해야 하는가. * 대부분 이 답변에선 면접관님들도 무시하라고 할 것이다..

5. 변수명은 어떻게 지정할것인가.
✅ 입력은 num1, num2 등으로 할 것인지 출력은 sum으로 할 것인지.
```

### II. 구체적 예시 탐색

---

- **문제**를 받았다면 이와 비슷한 예시를 탐색한다.
  1. I. 간단한 예시로 시작한다.
  2. II. 쉬운 예시와 비슷한 예시로 부터 복잡한 예시 순으로 2~3개의 예시를 탐색한다.
  3. III. 빈 입력값이 들어갈 수 있는가.
  4. IV. 잘못된 입력값이라면 어떻게 되는가.

#### 풀이 예시

---

- 구체적 예시 탐색 I ~ V 에 대한 답변을 정리한다.

```js
ex. 문자열을 입력값으로 받고 각 문자의 길이를 반환하는 함수 작성하기.

1. 간단한 예시로 시작한다.
stringCount("aaaa") // {a:4} 문자열 길이 출력
stringCount("hello") // {h:1, e:1, l:2, o:1} 각 알파벳 갯수를 합산

2. 복잡한 예시로 진행한다.
"my phone number is 421512" // 공백은 포함되어야하는가
"Hello hi" // 대소문자 구분없이 H, h 둘다 카운트하는가 아니면 개별로 카운트 하는가.

3. 누군가 stringCount() 을 입력하고 매개변수를 지정하지 않았다면 무엇을 반환할 것인가.
4. 누군가 문자열이 아닌 숫자나 객체를 매개변수로 한다면 어떻게 처리할 것인가.
```

### IV. 문제의 단순화

---

- **주석**으로 정리한 후 코드를 작성하려고 할때 모든 부분을 한번에 작성하려고 하면 어려움에 막힐 수 있다.
- 모든 부분을 작성하기 어렵다고 느낀다면 단순한 문제를 먼저 해결하고 해당 부분을 마지막으로 작성해야한다.
- 위 문제를 보면 **영문자/숫자를 판별**하는 것이 가장 어려운 부분으로 꼽을 것이다. 그 이전에 단순한 문제부터 해결해야한다.

#### 풀이 예시

---

- 먼저 코드 작성시 단순한 문제부터 해결한다.

```js
ex. 문자열을 입력값으로 받고 각 문자의 길이를 반환하는 함수 작성하기.

function charCount(str) {
    // 반환할 객체를 생성한다.
    var result = {};

    // 매개로 받은 문자열을 순회한다.
    for(var i = 0 ; i < str.length ; i++){

        // 대문자의 경우 한 번에 한 문자씩 바꾸는가 아니면 문자열 전체를 한번에 바꾸는가. > 면접관에게 질문이 필요하다.
        var char = str[i].toLowerCase()
        // 숫자/문자/대소문자 를 무시하고 정확한 문자가 객체에 있는지 확인한다.
        // 영숫자만 이 다음단계로 진행한다.
        if(/[a-z0-9]/.test(char)){
            if(result[char] > 0){

                // 이미 객체가 있는 경우
                // 문자를 순회하면서 문자가 객체에 키로써 있다면 1을 더한다.
                result[char]++;
            } else {
                // 문자를 순회하면서 문자가 객체에 키로써 없다면 키 생성 후 값을 1로 저장한다.
                result[char] = 1;
            };
        }
    }
    // 루프 적용하여 세부 코드 작성 후 객체 반환
    return result;
}

charCount("hello");
> {h: 1, e: 1, l: 2, o: 1}
```

### V. 리펙터

---

- 문제를 해결하는데 성공하고 시간이 남아있다면 코드를 향상시킬 수 있는지 되돌아보고 리펙토링을 진행한다.

#### 풀이 예시

---

- 완성된 코드를 보고 리펙토링을 진행한다.

```js
// 이전 코드
function charCount(str) {
    var result = {};

    // for 문 대신 for-of 루프를 이용하면 각 문자가 즉시 출력되므로 구문을 변경하는것이 좋다.
    for(var i = 0 ; i < str.length ; i++){
        // 소문자로 변경하는 위치가 타당한가?
        var char = str[i].toLowerCase()
        // 정규식은 이런 패턴을 찾아내는데 좋지만 성능을 바란다면 다른 방법도 강구하는것도 좋다.
        if(/[a-z0-9]/.test(char)){
            // 가장 단순한 if 문으로 문제 해결을 했을 것. 이 부분을 리펙토링한다.
            if(result[char] > 0){
                result[char]++;
            } else {
                result[char] = 1;
            };
        }
    }
    return result;
}

// 리펙터
function charCount(str) {
    var result = {};
    for(var char of str){
        if(isAlphaNumeric(char)){
            var char = char.toLowerCase()
            // 값이 참 이라면 1을 추가. 없다면 값을 1로 설정한다.
            result[char] = ++result[char] || 1;
        }
    }
    return result;
}

// 영숫자 정규식을 대체할 함수 위 정규식 보다 55% 성능 향상 효과가 있음.
function isAlphaNumeric(char) {
```
